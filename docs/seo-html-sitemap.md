# HTML Sitemap editorial

## Objetivo
Usar `/mapa-del-sitio` como hub indexable de enlazado interno para reforzar URLs ya valiosas dentro de mariachis.co.

No es un repositorio exhaustivo de combinaciones. Es una capa curada para:

- descubrimiento por ciudad
- hubs por evento
- zonas con oferta real
- recursos y accesos clave del ecosistema

## Bloques incluidos

1. Ciudades principales
2. Eventos principales
3. Zonas destacadas agrupadas por ciudad
4. Landings ciudad + evento destacadas
5. Servicios principales como bloque secundario
6. Recursos
7. Para mariachis

## Criterio editorial

Los umbrales son orientativos. No se deben tratar como reglas ciegas cuando perjudiquen un hub comercialmente valioso, pero sí como el piso mínimo para evitar spam SEO.

### Ciudad

- referencia: 6 anuncios publicados
- referencia: 3 perfiles distintos
- flexibilidad: también puede entrar una ciudad con volumen claramente superior aunque la diversidad quede apenas por debajo

### Evento nacional

- referencia: 8 anuncios publicados
- referencia: presencia en 3 ciudades

### Ciudad + evento

- referencia: 4 anuncios publicados
- referencia: 2 perfiles distintos

### Zona

- referencia: 3 anuncios publicados
- referencia: 2 perfiles distintos

### Servicio

- referencia: 8 anuncios publicados
- referencia: presencia en 2 ciudades
- este bloque se mantiene como exploración secundaria mientras el producto siga apoyándose en query params

## Qué no debe entrar

- páginas con oferta pobre o casi vacía
- combinaciones creadas solo por volumen de formularios
- taxonomías duplicadas
- landings que no aporten nada distinto a otra URL mejor

## Footer resumido

Mantener solo cuatro grupos:

- ciudades
- eventos
- recursos
- para mariachis

El footer no debe volver a crecer como una lista infinita. El HTML sitemap es el hub expandido; el footer solo debe anunciarlo.
